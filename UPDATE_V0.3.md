# Edge TTS Web v0.3 更新说明

v0.3 聚焦产品化底座，目标是在不大幅改变使用方式的前提下，提升安全性、稳定性、错误可诊断性和后续迭代可测试性。

## 核心更新

### 1. 受控文件 token 下载

- 单条转换接口不再把本机临时文件路径返回给前端。
- `/convert` 返回 `file_id`，前端通过 `/download/<file_id>` 下载音频。
- 旧的 `/download?path=...` 路径式下载入口不再读取任何文件，只返回结构化错误。
- 下载文件名会经过清理，避免路径分隔符和非法字符进入响应。

### 2. 音色白名单校验

- 单条转换和批量转换都会校验 `voice` 是否存在于当前可用音色列表。
- 未知音色统一返回 `invalid_voice` 错误，避免不可控外部请求。

### 3. 应用专用临时目录

- 所有生成的 MP3 都进入应用专用临时目录。
- 清理逻辑只会删除受控目录内的音频文件。
- 启动时和请求周期内会触发过期结果清理。
- 单条结果和批量任务都纳入临时文件引用保护，避免误删正在使用的音频。

### 4. 统一 API 错误响应

错误响应统一为：

```json
{
  "error": {
    "code": "invalid_voice",
    "message": "不支持的语音角色，请从列表中选择可用音色。"
  }
}
```

前端已适配新错误结构，并兼容旧字符串错误格式。

### 5. 基础自动化测试

新增 `pytest` 测试覆盖：

- 语速标准化和非法语速拒绝
- 下载文件名清理
- TXT 编码识别
- 单条转换返回 `file_id` 而不是本机路径
- token 下载和旧路径下载拒绝
- 未知音色拒绝
- 批量转换基础错误
- 受控临时文件删除和过期清理
