# Edge TTS Web v0.4 更新说明

v0.4 聚焦长文本任务体验，把单条文本转换从同步请求升级为可观察、可取消、可重试的异步任务，并为超长文本加入自动分段处理。

## 核心更新

### 1. 单条转换异步任务化

- `/convert` 不再阻塞等待音频生成完成。
- 提交文本后会立即创建文本任务并返回 `job_id`。
- 前端通过 `/jobs/<job_id>` 轮询任务状态。
- 任务返回总片段数、已完成数、成功数、失败数、耗时和预计剩余时间。

### 2. 长文本自动分段

- 单条文本上限提升到 200000 字符。
- 后端按自然断点把文本拆成片段，默认每段最多 5000 字。
- 分段优先级：换行、句号、问号、感叹号，然后退到分号、逗号等弱断点。
- 单个片段失败不会阻断其他片段继续处理。

### 3. 分段下载降级方案

- 单片段成功时下载 MP3。
- 多片段成功时下载 ZIP，包含按顺序命名的分段 MP3。
- 该方案避免引入 ffmpeg 等额外音频合并依赖，后续可在 v0.5+ 再升级为完整 MP3 合并。

### 4. 任务取消

- 新增 `/jobs/<job_id>/cancel`。
- 用户可以在任务处理中请求取消。
- 当前正在生成的片段无法强制中断，但后续待处理片段会标记为已取消。
- 排队任务可立即整体取消。

### 5. 失败片段重试

- 新增 `/jobs/<job_id>/items/<item_id>/retry`。
- 支持对失败或已取消片段单独重试。
- 前端提供全局“重试失败片段”和单片段“重试”按钮。

### 6. 前端进度体验

- 单条转换页面展示任务进度条。
- 展示每个片段的状态、字数、文件大小、尝试次数和错误信息。
- 完成后展示下载区域。
- 支持任务取消和失败片段重试。

## 相关接口

```text
POST /convert
GET  /jobs/<job_id>
POST /jobs/<job_id>/cancel
POST /jobs/<job_id>/items/<item_id>/retry
GET  /jobs/<job_id>/download
```

## 相关文件

- `app.py`：文本任务模型、分段、取消、重试和任务下载
- `templates/index.html`：单条任务进度、取消和重试交互
- `static/css/style.css`：任务状态样式
- `tests/test_app.py`：v0.4 自动化测试
- `README.md`：v0.4 使用说明

## 已知取舍

- v0.4 使用 ZIP 作为多段结果的下载格式，暂未做 MP3 合并。
- 取消任务不能强制中断当前正在请求 edge-tts 的片段，只会阻止后续片段继续处理。
- 文本任务仍保存在内存中，重启后不会恢复；持久化任务可作为后续版本继续推进。
